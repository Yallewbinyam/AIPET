# AIPET — Responsible Use Policy

## Legal and Ethical Requirements

AIPET is a penetration testing framework designed
exclusively for **authorised security assessments**.

### You MUST have explicit written permission before:
- Scanning any network or device
- Running any attack module against any target
- Extracting or analysing any firmware image

### Unauthorised use may violate:
- Computer Misuse Act 1990 (UK)
- Computer Fraud and Abuse Act (USA)
- Equivalent laws in your jurisdiction

---

## Permitted Uses

- Testing devices and networks you own
- Authorised penetration testing engagements
- Academic research in isolated lab environments
- Security awareness and education
- CTF (Capture The Flag) competitions
- Testing against the included virtual lab targets

---

## Prohibited Uses

- Scanning networks without written authorisation
- Testing devices you do not own or have permission to test
- Using findings to exploit or damage systems
- Targeting critical infrastructure
- Any use that violates applicable law

---

## Responsible Disclosure

If you discover a vulnerability using AIPET:

1. Do not exploit beyond what is necessary to demonstrate
2. Report to the affected organisation through their
   responsible disclosure programme
3. Allow reasonable time for remediation
4. Follow NCSC and CISA responsible disclosure guidelines
