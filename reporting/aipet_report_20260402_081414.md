## Recommendations

The following recommendations are prioritised by severity. Address Critical items immediately before moving to High severity items.

### 🚨 Critical Priority

1. **Disable anonymous MQTT access** — Enforce authentication on all MQTT brokers. No device should connect without credentials.

2. **Remove hardcoded credentials from firmware** — Issue firmware update. All devices running affected firmware are vulnerable.

3. **Revoke and replace shared private keys** — Every device sharing the same key is vulnerable to traffic decryption and impersonation.

4. **Disable Telnet** — Replace with SSH on all devices. Telnet exposes credentials on the network.

### 🔴 High Priority

5. **Change all default web interface credentials** — Each device must have unique, strong credentials.

6. **Restrict admin interface access** — Admin panels should not be accessible without authentication. Apply IP whitelisting where possible.

7. **Update vulnerable firmware components** — OpenSSL 1.0.x and OpenSSH < 7.4 have known critical vulnerabilities. Update immediately.

### ⚠️ Medium Priority

8. **Implement CoAP authentication** — Use DTLS for CoAP communication. Restrict resource access to authorised clients.

9. **Implement replay protection** — Add nonce or timestamp validation to CoAP and MQTT message processing.

10. **Add HTTP security headers** — Implement X-Frame-Options, Content-Security-Policy, and X-Content-Type-Options on all web interfaces.

### ℹ️ General Best Practices

- Implement network segmentation — IoT devices should be on isolated VLANs
- Enable firmware automatic updates where possible
- Monitor all IoT device communications for anomalous behaviour
- Conduct regular penetration testing as devices and firmware are updated
- Subscribe to CVE notifications for all IoT device models and firmware versions
